Создай десктопное приложение "Бот Майнкрафт" на базе Electron/Tauri со следующей архитектурой:

=== ЯДРО: OLLAMA + MINEFLAYER ===
1. Интеграция с Ollama (localhost:11434):
   - При первом запуске проверять наличие Ollama, если нет — показать инструкцию по установке
   - Встроенный каталог моделей (Llama 3, Mistral, Gemma, CodeLlama, DeepSeek и др.)
   - Кнопка "Скачать модель" — вызывает ollama pull <model>
   - Отображение статуса: скачано/не скачано, размер, VRAM/RAM требования
   - API-эндпоинт для чата: POST /api/generate с параметром model

2. Интеграция с Mineflayer (Node.js):
   - Поддержка только Minecraft Java Edition
   - Бот подключается к серверу по IP/порту
   - Авторизация: офлайн-режим (без лицензии) или с лицензией Mojang

=== ИНТЕРФЕЙС (GUI) ===

ЛЕВАЯ ПАНЕЛЬ — Управление ИИ:
- Переключатель режима: "Локальная нейросеть" / "API-ключ"
  • При выборе "Локальная" — показывается список моделей из Ollama
  • При выборе "API-ключ" — поле ввода ключа (OpenAI, Claude и др.)
- Каталог моделей с кнопками "Скачать / Удалить / Запустить"
- Индикатор загрузки модели в память (RAM/VRAM usage)

ЦЕНТРАЛЬНАЯ ПАНЕЛЬ — Мониторинг бота:
- Полоски HP, Голод, Опыт, Броня (живое обновление из Mineflayer)
- Инвентарь: сетка 9×4 с иконками предметов, подсветка активного слота
- Координаты (X, Y, Z) и биом
- Статус: "Оффлайн" / "Подключение" / "В игре" / "Автономный режим"

ПРАВАЯ ПАНЕЛЬ — Чат и управление:
- Чат-окно (история сообщений):
  • ОФФЛАЙН-ЧАТ: когда бот не на сервере — можно писать вопросы ИИ, он отвечает через Ollama
  • ИГРОВОЙ ЧАТ: сообщения из Minecraft + ответы бота в чат сервера
- Поле ввода сообщения + кнопка "Отправить"
- Галочка "Автоответ в чате" — бот сам пишет ответы на сообщения игроков

ПАНЕЛЬ УПРАВЛЕНИЯ (под чатом):
- Кнопка "Сменить ник" — поле ввода нового имени бота
- Кнопка "Остановить действие" — прервать текущую задачу (добыча, движение)
- Кнопка "Остановиться" — бот перестаёт двигаться, отменяет pathfinding
- Кнопка "ВЫЖИВАЛЬЩИК" — запуск автономного режима выживания:
  • Бот самостоятельно добывает дерево → крафтит верстак → инструменты
  • Ищет еду (убивает животных, собирает яблоки, сеет пшеницу)
  • Строит укрытие на ночь
  • Ищет портал в Незер → добывает обсидиан → активирует портал
  • В Незере ищет крепость → активирует портал Края
  • В Краю ищет дракона → убивает его (строит башню, ломает кристаллы)
  • ВСЕ ДЕЙСТВИЯ через ИИ: LLM принимает решения на основе состояния мира (инвентарь, HP, окружение)

ПРОФИЛЬ БОТА (отдельное окно/вкладка):
- Текстовое поле "Системный промт" — пользователь вставляет свой промт, который бот читает перед каждым действием
- Примеры: "Ты осторожный игрок, избегай лавы", "Ты агрессивный PvP-бот"
- Промт передаётся в Ollama как SYSTEM-инструкция при каждом запросе

=== ЛОГИКА РАБОТЫ ===
1. Пользователь выбирает модель Ollama → скачивает → запускает
2. Вводит IP сервера, порт, ник бота
3. Подключается — Mineflayer создаёт экземпляр бота
4. Все события (health, food, chat, spawn, death) передаются в GUI через WebSocket
5. ИИ получает контекст: "HP: 10/20, Голод: 15/20, Инвентарь: [дерево x12, камень x5], Координаты: 120 64 -340, Видишь: [корова, дерево, скелет]"
6. ИИ отвечает JSON-командой: {"action": "attack", "target": "skelton"} или {"action": "craft", "item": "wooden_pickaxe"}
7. Mineflayer выполняет команду, результат возвращается в ИИ для следующего шага

=== ТРЕБОВАНИЯ ===
- Без обязательных API-ключей (локальная работа через Ollama)
- Опциональная поддержка API-ключей (переключатель в интерфейсе)
- Кроссплатформенность: Windows, Linux, macOS
- Автономный режим "Выживальщик" работает полностью через LLM-рассуждения
- Чат работает и без подключения к серверу (оффлайн-режим)


=== ДОПОЛНИТЕЛЬНЫЕ ТРЕБОВАНИЯ ===
- Самодостаточное приложение: единый EXE-файл, запускается без установки дополнительных зависимостей
- Ollama устанавливается автоматически при первом запуске (встроенная автоустановка)

=== МУЛЬТИБОТЫ ===
- Возможность запускать НЕСКОЛЬКО ботов одновременно (каждый в отдельном процессе)
- У каждого бота СВОИ настройки:
  • Ник, сервер (IP:порт), версия Minecraft
  • Выбор модели ИИ (можно разные LLM на разных ботах)
  • Персональный системный промт
  • Прокси (свой или общий)
  • Авто-логин / авто-регистрация (вкл/выкл)
- Кнопка "Отключить ИИ" на каждом боте — бот продолжает работать, но без LLM (только базовые команды Mineflayer)
  • Позволяет запускать 60+ ботов без нагрузки на GPU/CPU
  • Каждый пользователь сам решает, скольким ботам включить ИИ (1, 2, 8, 60)

=== ОБЩИЙ ПАРОЛЬ ===
- В глобальных настройках приложения: поле "Пароль для регистрации/логина"
- Пользователь вводит пароль ОДИН РАЗ (например: 1111)
- ВСЕ боты используют этот же пароль для /register и /login на любых серверах
- Пароль хранится в зашифрованном виде в конфиге приложения

=== СИСТЕМА ОБЪЕДИНЕНИЯ БОТОВ (КООПЕРАЦИЯ) ===
- Боты общаются между собой ЧЕРЕЗ ПК ПОЛЬЗОВАТЕЛЯ (локальный WebSocket/IPC), без определения сервера
- Каждый бот отправляет свои данные (координаты, инвентарь, задача, статус) в локальный координатор
- Координатор распределяет задачи: "Бот А — добывает дерево, Бот Б — ищет еду, Бот В — строит укрытие"
- Боты сами решают, кто что делает, как настоящие игроки в команде
- Общий чат ботов: видны сообщения всех ботов между собой (планирование, запросы помощи)

=== ПРОКСИ ===
- Поддержка ВСЕХ типов прокси: SOCKS4, SOCKS5, HTTP, HTTPS
- Поле ввода: IP:порт, логин/пароль (если нужно)
- Проверка прокси перед подключением (кнопка "Проверить")
- Можно задать общий прокси для всех ботов или индивидуальный для каждого

=== АВТО-ЛОГИН / АВТО-РЕГИСТРАЦИЯ ===
- Бот мониторит чат сервера на наличие команд регистрации/логина
- Если в чате появляется: "Зарегистрируйтесь /register", "Войдите /login", "Напишите пароль" и т.д.
- Бот автоматически выполняет команду с ОБЩИМ паролем из глобальных настроек
- Поддержка разных форматов: /register пароль, /login пароль, /reg пароль пароль

=== ПРОХОЖДЕНИЕ КАПЧ ===
- Бот умеет проходить ВСЕ типы капч:
  • Текстовые в чате: "Напишите 123", "Решите 5+3" — парсинг + ответ
  • Капчи-картинки: OCR через встроенную библиотеку (Tesseract.js) + отправка в LLM для распознавания
  • ReCaptcha/аналоги: передача скриншота в LLM, получение инструкций (кликни на X, введи Y)
  • Вопрос-ответ: "Какой цвет у неба?" — LLM генерирует ответ
- Режим "Ручная помощь": если бот не справился — всплывающее окно для пользователя

=== КАТАЛОГ МОДЕЛЕЙ (расширенный) ===
- Кнопка "Загрузить свою модель" — пользователь указывает путь к GGUF-файлу, приложение регистрирует его в Ollama
- Индикатор VRAM/RAM использования для каждой модели
